# PolicyAnalyzerPro 项目规则

## 核心原则

### 1. 离线模式不能被破坏
- 离线分析是项目的核心功能，必须始终保持可用
- 在线和混合模式是可选增强，不能影响离线模式的稳定性
- 当在线/混合模式不可用时，必须自动回退到离线模式
- 离线模式不依赖任何外部网络服务

### 2. UI 主线程不能阻塞
- 所有耗时操作必须在后台线程执行
- 使用 QThread + ProcessPoolExecutor 进行异步处理
- 保持界面响应，提供进度反馈
- 支持任务取消操作

### 3. 在线能力必须可关闭
- 在线分析功能可以通过配置完全禁用
- 当在线功能未启用时，UI 上要有明确标识
- 不能误导用户以为在线功能可用
- 在线功能缺失时，不能抛出技术性异常

### 4. 新状态必须可观测
- 所有状态变化必须在界面上有视觉反馈
- 分析模式、在线状态、依赖状态必须清晰可见
- 状态提示文字要非技术化，方便普通用户理解
- 提供工具提示（tooltip）解释状态含义

## 分析模式规范

### 模式定义
- **离线分析 (offline)**: 完全本地执行，不依赖网络
- **在线分析 (online)**: 使用在线 LLM 服务进行分析
- **混合分析 (hybrid)**: 结合本地预处理和在线增强分析

### 模式切换规则
1. 默认展示为离线模式
2. 用户可以自由切换三种模式
3. 切换模式时，当前分析按钮行为要跟着变化
4. 切换模式后，状态提示要实时更新

### 回退机制
- 在线模式不可用时，自动回退到离线模式
- 混合模式不可用时，自动回退到离线模式
- 回退时要给用户明确的提示信息
- 回退不能导致界面卡顿或异常

## 状态显示规范

### 在线状态
- **在线就绪 (ready)**: 绿色徽章，显示"在线就绪"
- **骨架就位 (skeleton)**: 蓝色徽章，显示"骨架就位"
- **待配置 (unconfigured)**: 黄色徽章，显示"待配置"
- **在线关闭 (disabled)**: 灰色徽章，显示"在线关闭"

### 依赖状态
- **环境正常 (ok)**: 绿色徽章，显示"环境正常"
- **依赖警告 (warning)**: 黄色徽章，显示"依赖警告"
- **依赖缺失 (error)**: 红色徽章，显示"依赖缺失"

### 状态交互
- 依赖状态徽章可点击，点击打开环境自检向导
- 在线状态徽章显示工具提示，解释当前状态
- 所有状态变化必须在界面上实时反映

## UI 布局规范

### 主界面结构
1. **标题栏**: 应用名称和副标题
2. **状态区**: 环境状态、分析模式、在线状态、依赖状态
3. **输入区**: 单篇/双篇/批量分析输入
4. **输出区**: 分析结果展示和导出

### 模式切换控件
- 位于状态区，使用下拉框选择
- 包含三个选项：离线分析（默认）、在线分析、混合分析
- 下拉框宽度最小 160px
- 切换时实时更新状态显示

### 状态徽章布局
- 分析模式下拉框 → 在线状态徽章 → 依赖状态徽章 → 状态说明文字
- 徽章之间间距 8px
- 徽章圆角 10px，内边距 4px 8px

## 开发规范

### 代码组织
- UI 代码放在 `ui/` 目录
- 业务逻辑放在 `core/` 目录
- 导入处理放在 `importers/` 目录
- 测试代码放在 `tests/` 目录

### 信号连接
- 在 `_connect_signals()` 方法中集中连接信号
- 使用 `clicked` 信号处理按钮点击
- 使用 `currentIndexChanged` 信号处理下拉框切换
- 自定义信号用于跨组件通信

### 状态更新
- 使用 `_update_*_status()` 方法更新状态显示
- 状态更新方法在初始化时调用一次
- 配置变化时重新调用状态更新方法
- 保持状态显示的一致性

## 测试规范

### 必须验证的场景
1. 离线模式正常工作
2. 在线模式可用时正确切换
3. 在线模式不可用时自动回退
4. 混合模式可用时正确切换
5. 混合模式不可用时自动回退
6. 模式切换时界面不卡顿
7. 单篇/双篇/批量分析在三种模式下都能正常工作

### 状态验证
- 验证所有状态徽章正确显示
- 验证状态工具提示正确显示
- 验证点击依赖徽章打开环境向导
- 验证状态颜色符合规范

2. 运行语法检查：`python -m py_compile <file>`
3. 运行相关测试：`python -m unittest <test_file>`
4. 启动应用验证：`python main.py`
5. 验证功能符合预期

## 注意事项

- 保持主窗口结构稳定，不做大改版
- 所有新增控件要符合现有 UI 风格
- 状态提示文字要简洁明了
